## 3. 关键验收项

### 3.1 ONLYOFFICE 在线编辑

- DOCX、XLSX、PPTX 均能生成类型正确的编辑器配置。
- 无更新权限的用户不能发起在线编辑。
- 编辑会话锁定文件，关闭或超时后释放锁。
- 状态 2/6 回调创建新版本，状态 4 不创建版本。
- 重复回调不会重复创建版本。
- 非 Document Server 同源下载地址会被拒绝。
- 同步目录文件禁止在线编辑，避免覆盖管理员维护的服务器源文件。

### 3.2 审批工作流

- 支持文档流程、发布、下载、借阅、外发和权限申请模板。
- 模板步骤可按密级、敏感标记、扩展名和文件大小设置条件。
- 不匹配步骤自动跳过；没有任何匹配步骤时拒绝提交。
- 审批中心可转交、前后加签、撤回和催办。
- 模板可配置处理时限，临期和逾期提醒只发送一次。
- 借阅、外发和权限申请通过后按有效期动态授权。

### 3.3 通知与运维

- 企业微信投递会记录渠道、状态、次数、最近错误和下次重试时间。
- 首次推送失败后可由管理员重试并恢复为已发送。
- 备份包包含脱敏账本、所有本地版本、文档附件和公告附件。
- 恢复演练校验 ZIP 路径、账本格式及所有本地引用文件，不覆盖生产数据。
- 运行状态实际探测 MySQL、ONLYOFFICE 和磁盘容量。
- MySQL 启动连接超过 4 秒时快速降级本地 JSON，避免服务长时间无监听。
- 告警覆盖登录失败、连续预览失败、同步失败、编辑失败、通知失败、存储不一致和磁盘不足。

## 6. 安全检查

- MySQL 密码、企业微信 Secret、ONLYOFFICE JWT、API Secret hash/salt 和登录票据不会进入备份明文。
- 配置查询接口只返回敏感值是否已配置，不回显原值。
- Git 工作区未加入数据库连接文件、SSH 文件或运行时备份文件。
- 编辑回调票据绑定会话、文件、版本和用户，并设置有效期。
- ONLYOFFICE 回调缓存地址只接受 `/cache/files/` 路径，并强制改写到管理员配置的 Document Server 来源，兼容隧道和反向代理且不放宽 SSRF 边界。
- 彻底删除文件或文件夹时同步清理本地版本文件和文档附件，避免产生孤儿文件和磁盘持续增长。
