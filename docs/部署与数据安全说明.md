# 文档管理平台部署与数据安全说明

## 1. 项目说明

本项目是企业内部 B/S 文档管理平台原型，包含 Vue 3 + Vite + Element Plus 前端，以及 Node.js + Express 后端。当前版本已经支持文档库、服务器目录同步、权限控制、版本记录、审计日志、消息未读标记、系统管理和远程 MySQL 账本存储。

## 2. 本地启动

```bash
npm install
cp .env.example .env
npm run dev:server
```

访问地址：

```text
http://localhost:3000
```

默认开发账号：

```text
admin / admin123
demo / user123
```

## 3. 构建与测试

```bash
npm run build
npm test
```

`npm test` 会使用独立临时目录和测试端口，不会修改正式运行账本。

生产部署步骤、PM2/systemd/Nginx 示例见：

```text
docs/生产部署说明.md
```

## 4. 数据库与账本存储

系统支持两种账本存储方式：

- 本地 JSON：默认使用 `backend/data/db.json`
- 远程 MySQL：配置保存到 `backend/data/storage.json`

远程 MySQL 采用快照表存储当前系统账本：

```text
document_platform_store
```

后台入口：

```text
系统管理 -> 数据库存储 -> 连接配置 / 同步账本
```

## 5. 敏感信息处理

以下文件和目录不会提交到 GitHub：

```text
backend/data/
backend/uploads/
backend/tmp/
backend/backups/
backend/quarantine/
.env
.env.*
*.pptx
*.docx
*.tgz
*.zip
```

`.env.example` 是可提交的模板文件，只能放占位符，不能写真实密钥或数据库密码。

其中 `backend/data/storage.json` 包含远程 MySQL 主机、数据库名、用户名和密码，必须只保存在部署服务器本地，不能提交到仓库。

`backend/data/db.json` 是本地账本快照，可能包含用户、目录、审计、文件路径等业务数据，也不能提交。

## 6. 远程 MySQL 初始化

生产或测试库需要提前创建数据库并授权：

```sql
CREATE DATABASE IF NOT EXISTS your_database DEFAULT CHARACTER SET utf8mb4;
GRANT ALL PRIVILEGES ON your_database.* TO 'your_user'@'%';
FLUSH PRIVILEGES;
```

实际用户名、密码、访问来源 IP 和数据库名以部署环境为准。

## 7. 迁移/同步流程

1. 启动系统，使用管理员账号登录。
2. 进入“系统管理”。
3. 打开“数据库存储 -> 连接配置”。
4. 填写远程 MySQL 连接信息并测试连接。
5. 保存配置。
6. 点击“同步账本”，把当前本地账本写入 MySQL。
7. 重启服务，确认“实际运行”为“远程 MySQL”。

## 8. 提交仓库前检查

提交前建议执行：

```bash
git status --ignored
git diff --cached --name-only
npm run build
npm test
```

确认没有以下内容进入暂存区：

```text
storage.json
db.json
uploads
backups
tmp
node_modules
dist
数据库密码
```
