# 文档管理平台生产部署说明

## 1. 部署目标

生产服务器只需要运行一个 Node.js 服务。前端先通过 `npm run build` 构建到 `dist/`，后端启动后会自动托管前端页面和 `/api` 接口。

## 2. 环境要求

- Node.js 20 LTS 或更高版本。
- 可访问的 MySQL 服务。
- 一块稳定的服务器文件目录或 NAS 挂载目录，用于保存上传文件、临时文件和本地运行配置。
- 如需公网或内网域名访问，建议使用 Nginx 做反向代理。

## 3. 首次部署

```bash
git clone git@github.com:YaphetS0903/documentmanagement.git /opt/document-platform
cd /opt/document-platform
npm install --omit=dev --cache .npm-cache
cp .env.example .env
npm run build
npm run production:check
npm start
```

修改 `.env` 时只填写当前服务器自己的路径和密钥，不要把真实 `.env` 提交到 GitHub。

推荐生产路径示例：

```text
DATA_DIR=/data/document-platform/data
UPLOAD_DIR=/data/document-platform/uploads
TMP_DIR=/data/document-platform/tmp
BACKUP_DIR=/data/document-platform/backups
QUARANTINE_DIR=/data/document-platform/quarantine
EXTERNAL_LIBRARY_ROOT=/data/company-documents
```

## 4. 数据库配置

MySQL 连接信息不写进 `.env.example`，而是在系统后台配置：

```text
系统管理 -> 数据库存储 -> 连接配置
```

保存后系统会把连接配置写入服务器本地的 `storage.json`。该文件包含数据库地址、账号和密码，必须留在服务器本地，不能提交到仓库。

正式发布前，“系统管理 -> 运行状态”中的 MySQL 必须显示正常，不能保留“降级 JSON”状态。MySQL 故障时的 JSON 降级用于保障服务可启动，不代表满足生产发布条件。

如果要换 MySQL：

1. 管理员进入“系统管理 -> 数据库存储”。
2. 点击“连接配置”。
3. 填写新的 MySQL 地址、端口、库名、用户名和密码。
4. 测试连接并保存。
5. 根据页面提示执行账本同步。

## 5. PM2 启动

项目已提供 `ecosystem.config.cjs`：

```bash
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

更新代码后：

```bash
git pull
npm install --omit=dev --cache .npm-cache
npm run build
pm2 restart document-platform
```

首次启动前执行 `npm run production:check`。生产环境必须使用至少 32 字符随机 `JWT_SECRET`；空库首次初始化还必须设置至少 12 字符的 `INITIAL_ADMIN_PASSWORD`，初始化完成后应删除该环境变量并在页面中再次修改管理员密码。

## 6. systemd 启动

也可以使用 `deploy/document-platform.service.example`：

```bash
sudo cp deploy/document-platform.service.example /etc/systemd/system/document-platform.service
sudo systemctl daemon-reload
sudo systemctl enable document-platform
sudo systemctl start document-platform
sudo systemctl status document-platform
```

正式使用前需要按服务器实际情况调整：

- `WorkingDirectory`
- `EnvironmentFile`
- `ExecStart`
- `User`
- `Group`

## 7. Nginx 反向代理

参考 `deploy/nginx.conf.example`，把 `server_name` 改成实际域名，并确认 `proxy_pass` 端口与 `.env` 里的 `PORT` 一致。

修改后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 8. 备份建议

至少备份以下内容：

- MySQL 数据库。
- `UPLOAD_DIR` 上传文件目录。
- `DATA_DIR/storage.json` 数据库连接配置。
- 如仍使用本地 JSON 账本，也要备份 `DATA_DIR/db.json`。

备份文件不能放进 GitHub 仓库。

## 9. 发布门槛

候选版和正式版发布前执行：

```bash
npm run release:check
```

生产服务器再执行：

```bash
npm run production:check
```

完整人工步骤见 `docs/正式版发布检查清单.md`。
