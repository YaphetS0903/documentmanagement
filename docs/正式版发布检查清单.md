# 文档管理平台正式版发布检查清单

目标版本：`1.0.0-rc.1` -> `1.0.0`

## 一、发布候选版

- [x] 四期 M1-M5 功能完成并形成验收报告。
- [x] DOCX、XLSX、PPTX 在远程真实环境完成编辑和新版本保存。
- [x] API 冒烟测试、生产构建、语法和 Git 差异检查通过。
- [x] 仓库不包含数据库密码、JWT、SSH 文件、运行数据和备份。
- [x] 生产环境禁止使用开发 JWT，登录页不展示或预填默认账号密码。
- [x] ONLYOFFICE 禁用插件自动更新并设置内存保护上限。

## 二、生产环境

- [ ] 正式 MySQL 连接成功，运行状态不得降级为 JSON。
- [ ] 修改管理员及演示账号默认密码，停用不需要的演示账号。
- [ ] 配置正式域名、HTTPS 证书、反向代理和可信访问范围。
- [ ] 配置随机 `JWT_SECRET`、ONLYOFFICE JWT 和企业微信凭据。
- [ ] 确认 `DATA_DIR`、`UPLOAD_DIR`、`BACKUP_DIR` 位于持久化磁盘并具备正确权限。
- [ ] 执行 `npm run production:check` 并通过。
- [ ] 验证平台、MySQL、ONLYOFFICE、磁盘和同步目录健康状态。

## 三、用户验收

- [ ] 管理员完成上传、预览、在线编辑、版本、权限和删除恢复流程。
- [ ] 普通员工完成可见范围、同步目录、未读提醒、借阅和审批流程。
- [ ] 企业微信真实账号完成成功投递、失败记录和人工重试。
- [ ] 桌面和手机视口无严重遮挡、横向溢出或控制台错误。

## 四、发布与回滚

- [ ] 发布前创建完整备份并通过恢复演练。
- [ ] 记录当前数据库、上传目录、配置和应用版本。

- [ ] 发布后完成健康检查和核心业务冒烟测试。
- [ ] 若出现阻断问题，恢复上一应用版本及发布前数据备份。
