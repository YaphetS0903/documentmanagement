# 文档管理平台 B/S 架构功能需求清单

## 1. 文档来源与结论

### 1.1 参考资料

- 《云管家文档平台操作指南.pptx》
- 《开始云-Kass产品培训资料.pptx》
- 《开始云（文档管理）数据接口规范.docx》

### 1.2 总体结论

现有软件是基于 KASS/开始云体系的企业文档管理平台，核心能力不只是文件上传下载，还包含：

- 企业公共文档库
- 个人网盘
- 文档分类与知识地图
- 文档版本管理
- 文件夹/文件级权限
- 消息通知、订阅、提醒
- 分享、发布、收藏
- 评论、评分、附件、关联文件
- 全文检索
- 用户、部门、角色管理
- 开放 API 与单点登录

但老系统明显存在年代特征：

- 依赖 IE 或客户端访问。
- 在线编辑/预览依赖 NTKO 控件和本机 Office。
- 接口签名和参数设计较老，部分 API 描述存在重复或不规范。
- Web、移动端、企业微信集成体验需要重新设计。

新系统建议按纯 B/S 架构重做，支持现代浏览器、移动端/企业微信访问、无插件预览与在线编辑，并保留老系统中已经沉淀的文档权限、版本、分类、检索、通知、开放接口等核心能力。

## 2. 新系统建设目标

### 2.1 业务目标

- 建立统一的企业文档管理平台，集中管理制度、体系文件、部门文件、知识文件、历史资料和公司大事记。
- 替代旧客户端、IE、NTKO 控件等老旧依赖。
- 支持总部、工厂、部门、角色、岗位等多层级权限管理。
- 支持文件全生命周期管理：创建、上传、修改、发布、审批、归档、搜索、使用、回收、审计。
- 支持企业微信/移动端快速访问，满足办公电脑和手机端使用场景。
- 为后续其他业务系统提供统一 API 和单点登录能力。

### 2.2 技术目标

- 采用 B/S 架构，浏览器直接访问。
- 支持 Chrome、Edge、Safari 等现代浏览器。
- 不依赖 IE、ActiveX、NTKO、本机 Office 插件。
- 文件预览、在线编辑、转换、全文索引由服务端或在线 Office 服务完成。
- 后端提供标准 REST API，支持统一鉴权、权限校验、审计日志。
- 文件存储与业务数据库解耦，支持本地存储、NAS、对象存储等扩展。

## 3. 用户与角色需求

### 3.1 用户类型

- 普通员工：查看授权文档、搜索文档、下载/预览、收藏、评论、订阅。
- 部门文档管理员：维护本部门文件夹、上传/更新文件、设置部分权限、管理版本。
- 体系/质量管理员：维护 TES 体系文件、表单、推进计划、知识资料。
- 公司管理员：维护公告、目录、分类、权限规则、用户组织、系统配置。
- 系统集成账号：通过 API 与第三方系统交互。

### 3.2 用户基础能力

- 用户名/密码登录。
- 企业微信入口登录。
- 单点登录登录。
- 查看个人信息。
- 修改密码。
- 修改头像。
- 设置默认工作路径。
- 语言设置。
- 退出登录。

## 7. 非功能需求

### 7.1 性能

- 普通目录列表 2 秒内返回。
- 普通搜索 3 秒内返回第一页结果。
- 大文件上传支持进度显示和失败重试。
- 支持并发用户数、文件总量、存储容量需要按实际规模评估。

### 7.2 安全

- 所有接口必须进行身份认证。
- 所有文件访问必须进行权限校验。
- 下载、预览、外链地址必须有有效期。
- 敏感操作必须记录审计日志。
- 支持密码策略、账号禁用、登录失败锁定。
- 支持 HTTPS 部署。
- 支持文件水印、防外泄策略。

### 7.3 可用性

- 支持数据备份和恢复。
- 支持文件存储备份。
- 支持索引重建。
- 关键服务应支持监控告警。

### 7.4 兼容性

- 支持 PC 浏览器。
- 支持手机浏览器。
- 支持企业微信内置浏览器。
- 不依赖 IE、ActiveX、NTKO 控件。

### 7.5 可维护性

- 提供后台配置能力。
- 提供清晰 API 文档。
- 提供操作日志和错误日志。
- 权限、分类、目录、属性应支持配置，不应写死在代码中。

## 9. 需要业务确认的问题

- 旧系统中的所有历史文件是否需要完整迁移，还是只迁移有效文件和当前版本。
- 历史版本、评论、日志、权限规则是否需要迁移。
- TES、董事会、公司管理目录是否按现有结构保留，还是借重做机会重新梳理。
- 各目录默认权限如何设定，特别是下载、打印、导出 PDF、外链、修改、删除权限。
- 在线编辑采用 OnlyOffice、WPS、微软 Office Online、LibreOffice 转换方案还是其他方案。
- 企业微信是只做入口，还是要做免登、组织同步、消息推送。
- 是否需要审批流，如文件发布审批、外发审批、归档审批。
- 外链分享是否允许开放到公司外部。
- 文档密级、保密水印、下载控制是否有硬性要求。
- 搜索范围是否包含图片 OCR、扫描件 OCR。
- 公司大事记是否需要图片/视频管理能力。
