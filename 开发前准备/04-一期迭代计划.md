# 一期迭代计划

## 1. 迭代节奏建议

建议一期按 6 个迭代推进，每个迭代 1 到 2 周，先形成可演示闭环，再逐步补齐复杂能力。

## 2. 迭代 0：项目初始化

目标：搭好开发地基。

任务：

- 创建前端 Vue 3 + TypeScript + Vite 项目。
- 创建后端 Spring Boot 3 项目。
- 配置本地 Docker Compose：数据库、Redis、MinIO、OpenSearch。
- 建立基础代码规范、分支规范、提交规范。
- 建立数据库迁移机制。
- 建立统一返回格式和异常处理。
- 建立登录态/JWT 基础能力。

交付：

- 本地可启动前后端。
- 能访问登录页。
- 后端健康检查接口可用。

## 3. 迭代 1：组织与认证

目标：完成用户、部门、角色和登录。

任务：

- 用户表、部门表、角色表、关联表。
- 管理员初始化脚本。
- 登录、登出、当前用户接口。
- 用户管理页面。
- 部门树页面。
- 角色管理页面。
- 前端路由守卫。
- 基础菜单和布局。

交付：

- 管理员可登录系统。
- 可维护用户、部门、角色。
- 普通用户能按账号登录。

## 4. 迭代 2：文档库主流程

目标：跑通文件夹和文件上传下载闭环。

任务：

- doc_node、file_version 表。
- 文件夹创建、重命名、移动、删除。
- 文件上传、批量上传。
- 文件下载。
- 文件列表。
- 目录树。
- 文件详情。
- MinIO 存储接入。
- 文件操作审计日志。

交付：

- 可创建企业文档目录。
- 可上传、查看、下载文件。
- 可维护基础文件夹结构。

## 5. 迭代 3：版本、锁定与预览

目标：补齐文档管理核心能力。

任务：

- 文件版本列表。
- 上传更新生成新版本。
- 版本描述。
- 版本下载。
- 版本回滚。
- 文件锁定/解锁。
- PDF/图片/文本预览。
- Office 转 PDF 预览。
- 预览状态展示。

交付：

- 文件可多版本管理。
- 用户可在线预览常见文件。
- 文件编辑前可锁定，避免并发冲突。

## 6. 迭代 4：权限与搜索

目标：上线前最关键的一轮，保证“看得见、拿得到、改得了”都受控。

任务：

- 权限规则表。
- 权限动作枚举。
- 权限继承计算。
- 文件/文件夹权限配置页面。
- 后端接口权限拦截。
- 目录列表按权限过滤。
- 下载/预览权限校验。
- OpenSearch 接入。
- 文件文本抽取。
- 文件名和内容搜索。
- 搜索结果权限过滤。

交付：

- 不同用户看到不同目录和文件。
- 无权限用户不能预览或下载。
- 用户可搜索自己有权访问的文件。

## 7. 迭代 5：工作台、消息、收藏和初始化目录

目标：把系统做成可给业务试用的形态。

任务：

- 工作台首页。
- 最新文件。
- 最近操作。
- 收藏文件。
- 消息中心。
- 分享/订阅/版本变更消息预留。
- 初始目录导入脚本。
- 董事会、TES、公司管理等目录模板。
- 文件增长统计基础图表。

交付：

- 业务用户登录后有工作台。
- 初始目录结构与旧系统主要目录一致。
- 常用文件可收藏，消息可查看。

## 8. 迭代 6：联调、测试、部署

目标：准备试运行。

任务：

- 关键流程联调。
- 权限测试。
- 文件上传下载压力测试。
- 搜索准确性测试。
- 审计日志核对。
- 部署脚本。
- Nginx 配置。
- 数据库备份方案。
- 用户操作手册初稿。

交付：

- 测试环境可用。
- 一期功能可试运行。
- 有部署和备份说明。

## 9. 验收用例清单

### 9.1 登录与组织

- 管理员可以登录。
- 普通用户可以登录。
- 禁用用户不能登录。
- 用户可以被加入部门和角色。

### 9.2 文档库

- 有权限用户可以创建文件夹。
- 同级目录下不能创建同名文件夹。
- 用户可以上传单个文件。
- 用户可以批量上传文件。
- 用户可以下载有权限的文件。
- 用户不能下载无权限文件。
- 用户可以移动、复制、重命名有权限文件。

### 9.3 版本

- 上传更新后产生新版本。
- 历史版本可以预览和下载。
- 历史版本可以回滚。
- 版本记录包含操作者、时间、大小、描述。

### 9.4 权限

- 部门权限生效。
- 角色权限生效。
- 用户单独授权生效。
- 子目录继承权限生效。
- 权限取消后立即失效。

### 9.5 搜索

- 文件名可搜索。
- 文件内容可搜索。
- 搜索结果高亮。
- 无权限文件不出现在搜索结果中。

### 9.6 审计

- 登录记录被保存。
- 上传、下载、预览、删除被保存。
- 权限变更被保存。
- 日志可按时间和操作人查询。

## 10. 开发分工建议

- 前端 1：布局、登录、工作台、用户组织页面。
- 前端 2：文档库、上传下载、预览、权限配置。
- 后端 1：认证、用户组织、权限模型。
- 后端 2：文件存储、版本、预览、搜索。
- 测试/产品：验收用例、权限矩阵、目录初始化、试运行反馈。

如果团队较小，可以先由 1 名前端 + 1 名后端完成迭代 0 到迭代 2，再补人处理权限、搜索、预览。
