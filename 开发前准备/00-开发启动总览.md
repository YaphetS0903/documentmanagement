# 文档管理平台开发启动总览

## 1. 当前目标

基于已整理的《文档管理平台 B/S 架构功能需求清单》，启动新一代文档管理平台开发。新平台定位为企业内部 B/S 文档管理系统，替代旧 KASS/开始云客户端与 IE/NTKO 控件模式。

## 2. 一期建设原则

- 先做可用主干，再做增强能力。
- 先保证权限、文件、版本、检索、审计等核心能力正确。
- 目录、分类、权限、属性全部配置化，不写死在代码中。
- 文件访问必须经过后端授权，不能暴露真实存储路径。
- 所有敏感操作保留审计日志。
- 前端按现代浏览器和企业微信内置浏览器适配，不再兼容 IE。

## 3. 推荐一期范围

### 3.1 一期必须完成

- 用户登录与基础账号体系。
- 部门、角色、用户管理。
- 企业公共文档库。
- 文件夹创建、重命名、移动、删除。
- 文件上传、批量上传、下载、批量下载。
- 文件在线预览。
- 文件版本管理。
- 文件锁定/解锁。
- 文件/文件夹权限管理。
- 全文检索。
- 消息中心。
- 收藏。
- 审计日志。
- 初始业务目录导入。

### 3.2 一期可选增强

- 企业微信免登。
- 企业微信消息推送。
- 在线编辑。
- 个人网盘。
- 分享/发布。
- 订阅/文件闹钟。
- 分类/知识地图。
- 扩展属性。

## 4. 推荐技术栈

### 4.1 前端

- Vue 3
- TypeScript
- Vite
- Element Plus
- Pinia
- Vue Router
- Axios

选择理由：企业后台类 B/S 系统成熟、开发效率高、国内团队接受度高，适合目录树、表格、表单、权限配置等密集操作界面。

### 4.2 后端

- Java 17
- Spring Boot 3
- Spring Security
- MyBatis Plus 或 JPA
- PostgreSQL 或 MySQL 8
- Redis
- OpenSearch/Elasticsearch
- MinIO 或 NAS/对象存储

选择理由：文档管理平台偏企业级、权限复杂、审计要求高，Java/Spring Boot 更适合长期维护和内网部署。

### 4.3 文档处理

- OnlyOffice 或 WPS WebOffice：在线编辑。
- LibreOffice Headless：Office 转 PDF/HTML 预览兜底。
- PDF.js：PDF 浏览器预览。
- Apache Tika：文本抽取与全文索引。

### 4.4 部署

- 开发环境：Docker Compose。
- 生产环境：Linux 服务器 + Nginx + 后端服务 + 数据库 + Redis + 对象存储 + 搜索服务。
- 后续可扩展 Kubernetes，但一期不强依赖。

## 6. 一期开发顺序

1. 搭建前后端脚手架与本地开发环境。
2. 建立用户、部门、角色、权限基础模型。
3. 建立文件存储、文件夹、文件版本模型。
4. 实现登录、权限拦截、菜单和基础布局。
5. 实现文档库目录树、文件列表、上传下载。
6. 实现版本、锁定、审计日志。
7. 接入预览和全文检索。
8. 实现权限规则配置。
9. 实现消息、收藏、初始目录导入。
10. 做一期联调、测试、部署。

## 7. 开发前确认项

- 数据库最终选 PostgreSQL 还是 MySQL。
- 文件存储最终选 MinIO、NAS 还是本地磁盘。
- 在线编辑是否一期就做。
- 企业微信是否一期做免登和组织同步。
- 历史文件是否需要迁移。
- 历史版本、权限、日志是否需要迁移。
- 初始管理员账号和部门树来源。
- 内网访问域名、HTTPS 证书和部署服务器。
