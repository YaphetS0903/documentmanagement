# 二期开发规划

更新日期：2026-07-09

## 1. 二期定位

一期已经完成文档管理主流程，包括登录、组织角色、文档库、上传下载、版本、权限、搜索、消息、审计、个人网盘、远程 MySQL 账本配置和服务器目录同步。

二期重点不再继续堆基础按钮，而是围绕“安全管控、审批闭环、企业集成、试运行体验”做增强，让系统更接近企业内部正式使用状态。

## 2. 二期目标

- 敏感文档可识别、可限制、可追踪。
- 预览和下载行为更安全，支持水印和策略控制。
- 常见高风险操作进入审批流程。
- 企业微信集成具备配置入口和接口预留。
- 管理员能在后台统一配置安全、上传、集成和系统参数。
- 文档库日常使用体验更顺手，减少试用阶段的操作阻力。

## 5. 迭代 1 详细需求

建议二期第一轮从文件安全策略开始。

### 5.1 密级字段

文件和文件夹支持密级：

- 公开
- 内部
- 受限
- 机密

默认密级为“内部”。文件夹密级可以作为新上传文件的默认值，但文件本身可单独调整。

### 5.2 敏感标识

文件支持敏感标识：

- 普通文件
- 敏感文件

敏感文件在列表和搜索结果中只显示低调标记，避免页面过于杂乱。

### 5.3 安全策略

后台安全策略建议包含：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| enablePreviewWatermark | true | 是否启用预览水印 |
| enableDownloadWatermark | false | 是否预留下载水印能力 |
| blockSensitiveDownload | true | 是否禁止普通用户下载敏感文件 |
| allowAdminBypass | true | 管理员是否可跳过敏感下载限制 |
| logSensitiveAccess | true | 是否记录敏感文件访问日志 |
| watermarkTextMode | user | 水印内容模式 |

水印内容模式：

- `user`：当前用户姓名、账号、时间。
- `company`：固定公司名称。
- `custom`：自定义文字。

### 5.4 预览水印

预览水印要求：

- 颜色浅，不影响阅读。
- 倾斜平铺。
- 包含用户姓名或账号。
- 包含当前时间。
- PDF、图片、文本预览都应有覆盖层。
- 不改变原始文件。

### 5.5 下载限制

当 `blockSensitiveDownload=true` 且文件为敏感文件：

- 普通用户点击下载时提示“该文件受安全策略限制，仅允许在线预览”。
- 管理员可按 `allowAdminBypass` 策略决定是否允许下载。
- 被拦截的下载也要写入审计日志。

### 5.6 审计增强

敏感文件相关审计动作建议：

- `sensitive.preview`
- `sensitive.download`
- `sensitive.download.blocked`
- `sensitive.mark`
- `sensitive.unmark`
- `security-policy.update`

审计内容建议补充：

- 文件 ID。
- 文件路径。
- 密级。
- 是否敏感。
- 操作用户。
- IP。
- User-Agent。
- 操作结果。

## 9. 权限规则

安全策略不替代原有权限规则，而是在已有权限基础上追加限制。

判断顺序建议：

1. 判断用户是否有可见权限。
2. 判断用户是否有预览或下载权限。
3. 判断文件是否命中敏感策略。
4. 判断是否需要审批。
5. 记录审计日志。
6. 返回文件或拒绝原因。

## 10. 验收用例

### 10.1 文件安全策略

- 管理员可以把普通文件标记为敏感文件。
- 管理员可以设置文件密级。
- 普通用户可以看到自己有权限的敏感文件。
- 普通用户预览敏感文件时显示水印。
- 普通用户下载敏感文件时被安全策略拦截。
- 管理员是否可下载敏感文件受后台策略控制。
- 关闭“禁止敏感文件下载”后，普通用户在有权限时可以下载。
- 每次敏感文件预览、下载、下载拦截都有审计日志。

### 10.2 审批流程

- 普通用户可提交下载申请。
- 申请提交后审批人收到消息。
- 审批人可以同意或拒绝。
- 申请人可以看到审批结果。
- 审批通过后用户可以执行对应动作。
- 审批拒绝后用户不能执行对应动作。

### 10.3 配置中心

- 管理员能读取当前安全策略。
- 管理员能修改安全策略。
- 普通用户不能访问安全策略配置接口。
- 配置变更写入审计日志。

## 13. 风险与注意事项

- 水印是前端覆盖层，只能降低截图传播风险，不能作为绝对防泄密手段。
- 下载水印如果要写入 PDF/Office 文件本体，需要引入文件转换或加工服务，建议后续单独做。
- 企业微信集成依赖真实企业应用配置，开发阶段先做配置和接口预留。
- 审批流第一版不要做太复杂，先支持单级审批，后续再扩展多级审批。
- 安全策略必须叠加在权限规则之后，不能绕开现有权限体系。
