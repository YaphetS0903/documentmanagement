# 四期开发规划

更新日期：2026-07-11

## 1. 四期定位

四期主题为“在线协作、流程深化与可运营交付”。前三期已经完成文档主流程、安全审批、原版预览、全文检索和知识治理；四期重点让文档从“可管理、可查看”进入“可在线修改、可追踪协作、可稳定运营”的阶段。

四期首个里程碑是 ONLYOFFICE 在线编辑并自动保存为平台新版本。该能力复用现有版本、权限、锁定、审计和订阅体系，不另建一套文件存储。

## 2. 四期目标

- 有更新权限的用户可以在线编辑 `docx`、`xlsx`、`pptx`。
- 编辑期间文件自动锁定，其他用户仍可预览但不能同时发起编辑。
- 用户关闭编辑器后，Document Server 回传内容并生成新版本。
- 新版本保留创建人、时间、文件大小、MD5、正文索引和变更记录。
- 编辑失败、回调重复和网络中断不会破坏当前版本。
- 审批流程支持发布、借阅、下载和外发等多级场景。
- 消息中心覆盖版本更新、审批、复审和到期提醒，并为企业微信推送预留通道。
- 建立备份恢复、运行监控、异常告警和关键流程回归测试。

## 3. 功能范围

### 3.1 在线编辑与版本保存（M1）

- 文档库文件菜单增加“在线编辑”。
- 仅现代 Office 格式 `docx`、`xlsx`、`pptx` 支持编辑；旧格式继续原版只读预览。
- 发起编辑需要 `file:update` 权限，并遵守文件密码和锁定规则。
- 编辑配置使用 `mode=edit`，允许保存，不允许从编辑器直接下载或打印。
- 发起编辑时记录编辑会话并锁定文件。
- ONLYOFFICE 回调状态 `2` 或 `6` 时下载编辑结果，校验大小和扩展名后创建新版本。
- 状态 `4` 表示无修改关闭，只结束会话并释放锁。
- 同一会话的重复回调必须幂等，不重复生成版本。
- 保存成功后写入版本变更记录、审计日志并通知订阅用户。
- 编辑弹窗关闭后刷新文档列表和最近访问数据。

### 3.2 高级全文检索（M2）

- 增加搜索建议、最近搜索和常用条件。
- 支持文件类型、创建人、密级、分类、标签、更新时间和路径组合筛选。
- 搜索结果按标题、正文、路径和更新时间综合排序。
- Office、PDF 和代码类文件显示命中片段与关键词高亮。
- 管理员可查看索引失败任务并执行重建。

### 3.3 审批工作流增强（M3）

- 支持发布、借阅、下载、外发和权限申请流程模板。
- 支持串行多级审批、会签、或签、抄送和条件分支。
- 支持转交、加签、撤回、催办和审批超时提醒。
- 每次决策保留节点、处理人、意见、时间和结果。
- 审批通过后自动授予有期限的动作权限。

### 3.4 消息通知（M4）

- 统一上传、版本更新、评论、审批、复审和提醒消息。
- 支持未读聚合、批量已读和按类型筛选。
- 支持站内通知与企业微信推送策略。
- 推送失败可重试，并保留最近失败原因。

### 3.5 安全与运维（M5）

- 增加数据库和文件存储一致性检查。
- 支持备份任务、恢复演练和备份结果记录。
- 增加后端、MySQL、存储、ONLYOFFICE 和同步任务健康状态。
- 增加异常登录、连续预览失败、同步失败和磁盘不足告警。
- 关键配置变更继续写入审计日志，敏感值不回显。

## 4. 在线编辑数据模型

### 4.1 officeEditSessions

```json
{
  "id": "oes_xxx",
  "nodeId": "n_xxx",
  "baseVersionId": "ver_xxx",
  "userId": "u_xxx",
  "documentKey": "stable-key",
  "status": "active",
  "savedVersionId": null,
  "startedAt": "",
  "completedAt": null,
  "lastCallbackStatus": null,
  "lastError": ""
}
```

状态包括 `active`、`saved`、`closed`、`failed`。会话只保存业务标识，不保存用户登录令牌、ONLYOFFICE JWT 或回调密钥。

## 5. M1 API 清单

```text
POST /api/v1/files/:id/office-edit-session
POST /api/v1/office-edit/callback?ticket=...
GET  /api/v1/files/:id/office-edit-session
POST /api/v1/files/:id/office-edit-session/close
```

发起编辑接口返回同源 ONLYOFFICE API 地址、编辑器配置和会话摘要。回调接口由 Document Server 调用，使用专用短期票据鉴权，不使用浏览器登录令牌。

## 6. 安全约束

- 发起编辑必须具备 `file:update` 权限。
- 被其他用户锁定的文件返回 409；管理员可以按现有规则处理锁定。
- 回调票据绑定会话、文件、基础版本和用户，并设置有效期。
- 回调下载地址只接受 Document Server 同源地址或 `/cache/files/` 缓存路径；代理场景下缓存路径必须强制改写到系统配置的 Document Server 后再下载。
- 编辑结果必须通过文件策略大小和扩展名校验。
- 文件完全写入临时目录并计算校验信息后，才能切换当前版本。
- 重复回调、过期回调和已结束会话不得重复保存。
- JWT、数据库密码、SSH 信息和运行时票据不进入 Git。

## 7. M1 验收标准

- `docx`、`xlsx`、`pptx` 均能打开编辑器，标题和编辑器类型正确。
- 没有 `file:update` 权限的用户不能发起编辑。
- 编辑中的文件会显示锁定状态，第二名用户不能发起编辑。
- 有修改关闭后新增且仅新增一个版本，当前版本指向新版本。
- 新版本可下载、可预览、可搜索，并保留版本变更记录。
- 无修改关闭不新增版本，并正确释放锁。
- 重复回调不重复生成版本。
- 非 ONLYOFFICE 来源的回传下载地址被拒绝。
- Word、PPT、Excel 预览和编辑切换不会出现空白编辑器。
- `npm test`、`npm run build`、`git diff --check` 全部通过。

## 8. 开发顺序

1. 完成在线编辑会话模型和兼容初始化。
2. 实现编辑配置、回调票据和下载地址校验。
3. 实现回调保存、幂等、版本记录和解锁。
4. 增加文档库“在线编辑”入口和编辑弹窗。
5. 补充权限、锁定、保存、重复回调和失败回调测试。
6. 对三种现代 Office 格式做浏览器验收。
7. 开发高级全文检索。
8. 开发多级审批与统一通知。
9. 完成备份、监控、告警和部署验收。

## 9. 四期完成定义

在线编辑、检索增强、审批增强、消息通知、安全运维、自动化测试和真实环境验收全部完成后，四期才算完成。每个里程碑单独形成完成记录和验收报告，避免仅有页面或仅有接口即标记完成。
