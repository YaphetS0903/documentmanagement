# 开工前确认与下一步路线

更新日期：2026-07-08

## 1. 当前可开工基线

当前目录已经具备一个可运行的一期开发版，用于后续继续迭代和业务演示。

```bash
npm install --cache .npm-cache
npm run build
npm test
npm run start
```

访问地址：

```text
http://localhost:3000
```

默认账号：

```text
admin / admin123
demo / user123
```

## 2. 当前一期已覆盖主流程

- 登录、退出、当前用户。
- 用户、部门、角色基础管理。
- 企业文档库目录树。
- 文件夹和文件的新增、上传、批量上传、下载、批量下载、移动、复制、重命名、删除。
- 回收站恢复和彻底删除。
- 版本管理、上传新版本、版本回滚。
- 文件锁定和解锁。
- 权限规则和最终权限校验。
- 权限高级条件配置和生效校验。
- 文件名、正文、标签、分类、扩展属性搜索。
- 文本、PDF、图片预览，Office 文本抽取兜底。
- 收藏、消息、工作台、审计日志。
- 分享/发布、订阅、文件闹钟。
- 文件闹钟周期、提醒方式和结束时间维护。
- 附件和关联文件。
- 分类、标签、扩展属性、评论、评分。
- 分类树高级维护和扩展属性高级维护。
- 旧系统核心目录初始化。

## 3. 当前开发期技术栈

- 前端：Vue 3、Vite、Element Plus、Lucide 图标。
- 后端：Node.js、Express。
- 数据：JSON 文件数据库。
- 文件：本地上传目录。

这套技术栈适合快速验证一期业务闭环。生产环境建议继续按《系统架构与技术选型》迁移到关系数据库、对象存储、搜索服务和文档转换/在线编辑服务。

## 4. 生产化前必须确认

- 数据库：PostgreSQL 还是 MySQL。
- 文件存储：MinIO、NAS 还是其他对象存储。
- 在线编辑：OnlyOffice 还是 WPS WebOffice，是否纳入一期上线。
- 全文检索：OpenSearch 还是 Elasticsearch。
- 企业微信：是否做免登、组织同步、消息推送。
- 历史迁移：迁移文件范围、历史版本范围、权限和日志是否迁移。
- 部署：服务器、域名、HTTPS、备份策略。

## 5. 下一步推荐开发顺序

1. 做一次一期功能逐项验收，按需求清单逐条打钩。
2. 强化权限边界测试，尤其是搜索、预览、下载、分享后的授权。
3. 明确在线编辑和企业微信是否进一期正式上线。
4. 设计生产数据库表和迁移脚本。
5. 将 JSON 数据层替换为 PostgreSQL/MySQL。
6. 将本地文件目录替换为 MinIO/NAS。
7. 接入 OpenSearch/Elasticsearch 和文档正文抽取任务。
8. 准备测试环境部署脚本、备份方案和用户操作手册。
